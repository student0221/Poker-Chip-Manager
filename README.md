# 德州扑克筹码管理系统

局域网内的德州扑克筹码管理工具，支持单局兼容模式和多房间局域网对战模式。

## 使用

```bash
# 安装依赖
npm install

# 开发模式（前后端同时启动）
npm run dev

# 生产构建 + 启动
npm run build
npm start
```

Windows 下也可以直接运行 `start.bat`。脚本会打开房间大厅，并在终端显示同一 WiFi 手机可访问的局域网大厅地址。

## 推荐入口：局域网房间大厅

- **本机大厅**：`http://localhost:3000/#/rooms`
- **同 WiFi 设备大厅**：`http://<主机局域网IP>:3000/#/rooms`
- **房间页**：`http://<主机局域网IP>:3000/#/room/<房间码>`

大厅可以创建房间、输入房间码加入、查看本机房间和后端 UDP 发现到的局域网房间。房间页会显示邀请链接和二维码，手机扫码即可进入同一房间。

## 房间模式流程

1. 房主在 `/#/rooms` 创建房间，系统记录当前设备为房主。
2. 房主进入房间页，分享邀请二维码或房间链接。
3. 房主点击“开始比赛”。
4. 玩家用手机扫码进入房间页，填写昵称和入场筹码报名。
5. 比赛中房主可给玩家补筹码，其他设备会通过 Socket.io 自动刷新。
6. 房主点击“结束比赛”，玩家提交最终筹码。
7. 房主点击“执行清算”，所有设备刷新看到最终排名。
8. 房主可重置或解散房间。

## 兼容入口

旧单局入口仍然保留，用于原有玩法和兼容验证：

- **参与者页**：`http://<主机IP>:3000/#/`
- **管理后台**：`http://<主机IP>:3000/#/admin`

旧 API 会自动映射到默认房间，不会覆盖新房间 API。

## 局域网验收清单

先在主机上运行自动 smoke，确认服务能通过本机局域网地址完成基础房间流程：

```bash
npm run smoke:lan
```

在同一个 WiFi 下准备一台主机和至少两台手机/平板：

1. 主机运行 `start.bat` 或 `npm start`，确认终端显示 `http://<局域网IP>:3000/#/rooms`。
2. 手机 A 打开大厅地址或扫描房间二维码，确认能进入房间页。
3. 手机 B 使用同一个二维码进入同一房间。
4. 房主开始比赛，手机 A/B 分别报名，确认玩家列表在 500ms 左右自动刷新。
5. 房主给任一玩家补筹码，确认其他设备能看到变化。
6. 房主结束比赛，手机 A/B 提交最终筹码。
7. 房主执行清算，确认所有设备看到一致排名。
8. 另建第二个房间，确认同名玩家可出现在不同房间，但同一房间不能重复。
9. 用非房主设备尝试开始、结束、清算、重置或解散房间，应被拒绝。
10. 断开/刷新任意手机页面后重新进入，确认能拉取最新房间状态。

如果 UDP 自动发现没有出现房间，仍可通过 IP、房间链接或二维码加入；这是预期的降级路径。

## 技术栈

- React 18 + Vite + Tailwind CSS
- Express + sqlite3
- Socket.io 实时同步
- Node UDP 广播/监听局域网房间发现
- 响应式设计，支持手机浏览器
